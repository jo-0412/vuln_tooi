# U-03 계정 잠금 임계값 설정

## 1. 개요

- **항목 코드**: U-03
- **항목명**: 계정 잠금 임계값 설정
- **중요도**: 상
- **분류**: UNIX > 계정 관리

이 항목은 로그인 실패가 반복될 때 계정을 자동으로 잠그는 정책이 설정되어 있는지 점검한다.

---

## 2. 왜 이 항목이 필요한가

로그인 실패 횟수 제한이 없으면 공격자는 계정에 대해 무제한으로 비밀번호를 시도할 수 있다.  
이 경우 다음과 같은 공격에 취약해진다.

- 무차별 대입 공격(Brute Force)
- 사전 대입 공격(Dictionary Attack)
- 자동화된 계정 추측 공격
- 관리자 및 일반 사용자 계정 탈취 시도

이 항목의 목적은 **로그인 실패가 누적되면 계정을 잠가 반복 인증 시도를 차단하는 것**이다.

---

## 3. 점검 목적

로그인 실패 반복 시 계정을 잠가 brute force, dictionary 공격을 무력화한다.

---

## 4. 주요 확인 파일

### Debian 계열 우선 확인 파일
- `/etc/pam.d/common-auth`
  - 인증 실패 누적 정책 적용 여부 확인
- `/etc/pam.d/common-account`
  - 계정 잠금 후처리 또는 reset/account 적용 여부 확인

### 추가 확인 파일
- `/etc/security/faillock.conf`
  - `deny`, `unlock_time` 설정 확인
- `/etc/pam.d/system-auth`
  - RHEL 계열 호환 경로
- `/etc/pam.d/password-auth`
  - RHEL 계열 호환 경로
- `/etc/default/login`
- `/etc/security/policy.conf`
- `/etc/security/user`
- `/etc/default/security`

---

## 5. 확인 명령어

```bash
authselect current
grep -E 'deny|unlock_time' /etc/security/faillock.conf
grep -E 'pam_tally|pam_tally2|pam_faillock' /etc/pam.d/system-auth /etc/pam.d/password-auth
