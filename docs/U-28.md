# U-28 접속 IP 및 포트 제한

## 1. 개요

- **항목 코드**: U-28
- **항목명**: 접속 IP 및 포트 제한
- **중요도**: 상
- **분류**: UNIX > 네트워크 보안

이 항목은 허용된 호스트와 포트만 서비스에 접근할 수 있도록 제한하고 있는지 점검하는 항목이다.

---

## 2. 왜 이 항목이 필요한가

서비스가 모든 외부 IP에 그대로 노출되어 있으면,
공격자는 다음과 같은 경로로 시스템 접근을 시도할 수 있다.

- SSH 무차별 대입 공격
- FTP, Telnet 등 취약 서비스 접근
- 관리 포트 탐지 및 우회 접속
- 내부 전용 서비스의 외부 노출
- 방화벽 정책 미구성 상태 악용

이 항목의 목적은 **허용된 IP와 포트만 열어 외부 공격면을 줄이는 것**이다.

---

## 3. 점검 목적

허용된 호스트만 서비스에 접근하게 해 외부 공격면을 줄이기 위함

---

## 4. 주요 확인 파일

- `/etc/hosts.allow`
- `/etc/hosts.deny`
- `/etc/firewall/pf.conf`
- `/etc/ipf/ipf.conf`
- `/var/adm/inetd.sec`

1차 구현에서는 Linux 공통 기준으로 아래를 우선 확인한다.

- `/etc/hosts.allow`
- `/etc/hosts.deny`
- `/etc/ssh/sshd_config`

---

## 5. 확인 명령어

```bash
iptables -L
firewall-cmd --list-all
ufw status numbered
cat /etc/hosts.allow
cat /etc/hosts.deny
