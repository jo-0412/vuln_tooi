# U-37 Check crontab and at File Permissions

## 1. Overview

- **Item Code**: U-37
- **Item Name**: Check crontab and at File Permissions
- **Severity**: High
- **Category**: UNIX > File and Directory Management

This item checks whether cron and at related files are protected with proper ownership and permissions.

---

## 2. Purpose

The purpose of this check is to prevent ordinary users from abusing scheduled job mechanisms.

If cron or at related files are writable or executable by unauthorized users,
an attacker may create malicious scheduled jobs, establish persistence,
or damage the system.

---

## 3. Files and Directories to Check

### Executables

- `/usr/bin/crontab`
- `/usr/bin/at`

### Spool Directories

- `/var/spool/cron`
- `/var/spool/cron/crontabs`
- `/var/spool/cron/atjobs`
- `/var/spool/at`

### Control Files

- `/etc/cron.allow`
- `/etc/cron.deny`
- `/etc/at.allow`
- `/etc/at.deny`

---

## 4. Commands to Check

```bash
ls -l /usr/bin/crontab /usr/bin/at
ls -ld /var/spool/cron /var/spool/cron/crontabs /var/spool/cron/atjobs 2>/dev/null
ls -l /etc/cron.allow /etc/cron.deny /etc/at.allow /etc/at.deny 2>/dev/null
