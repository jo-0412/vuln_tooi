# U-64 주기적 보안 패치 및 벤더 권고사항 적용

## 1. 개요

- **항목 코드**: U-64
- **항목명**: 주기적 보안 패치 및 벤더 권고사항 적용
- **중요도**: 상
- **분류**: UNIX > 패치 관리

이 항목은 최신 보안 패치 적용과 벤더 권고사항 반영 체계가 존재하는지 점검하는 항목이다.

---

## 2. 왜 이 항목이 필요한가

운영체제가 최신 보안 패치를 적용하지 않으면,
이미 공개된 취약점이 장기간 방치될 수 있다.

이 경우 다음과 같은 문제가 발생할 수 있다.

- 공개 취약점 악용
- 원격 코드 실행 취약점 노출
- 권한 상승 취약점 방치
- 커널 및 패키지 취약점 누적
- 운영체제 안정성 저하

이 항목의 목적은 **패치 정책을 수립하고 주기적으로 패치를 관리하는 것**이다.

---

## 3. 점검 목적

최신 보안 패치 적용으로 시스템 안정성과 보안성을 확보하기 위함

---

## 4. 주요 확인 대상

고정 설정 파일만이 아니라 다음 항목을 함께 확인한다.

- 패치 정책 문서
- 정기점검보고서
- 작업계획서
- 패키지/커널 버전 정보
- 자동 업데이트 설정
- 최근 패치 이력

1차 구현에서는 Debian/Ubuntu 계열 기준으로 아래를 우선 확인한다.

- `/etc/apt/apt.conf.d/20auto-upgrades`
- `/etc/apt/apt.conf.d/50unattended-upgrades`
- `/var/log/apt/history.log`
- `/var/log/unattended-upgrades/unattended-upgrades.log`
- `apt list --upgradable`
- `hostnamectl`
- `uname -r`

---

hostnamectl
uname -r
apt list --upgradable
dpkg -l unattended-upgrades
grep -R "Unattended-Upgrade\|Update-Package-Lists" /etc/apt/apt.conf.d/
