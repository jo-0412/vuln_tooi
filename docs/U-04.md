# U-04 비밀번호 파일 보호

## 1. 개요

- **항목 코드**: U-04
- **항목명**: 비밀번호 파일 보호
- **중요도**: 상
- **분류**: UNIX > 계정 관리

이 항목은 비밀번호가 평문 또는 취약한 형태로 저장되지 않도록
비밀번호 저장 구조를 점검하는 항목이다.

---

## 2. 왜 이 항목이 필요한가

비밀번호가 `/etc/passwd` 같은 일반 계정 파일에 직접 저장되어 있으면,
파일 유출 또는 읽기 권한 오남용만으로도 계정 비밀번호 노출 위험이 커진다.

특히 다음과 같은 문제가 발생할 수 있다.

- 계정 파일 유출 시 비밀번호 해시 직접 노출
- 평문 또는 약한 형태 저장 시 즉시 계정 탈취
- 관리자 계정(root) 정보 유출
- 오프라인 크래킹 공격 위험 증가

이 항목의 목적은 비밀번호를 쉐도우 파일 등 별도 보호 저장소로 분리하고,
비밀번호 정보가 직접 노출되지 않도록 하는 것이다.

---

## 3. 점검 목적

비밀번호가 평문 또는 취약한 형태로 저장되지 않도록 보호하기 위함

---

## 4. 주요 확인 파일

### Linux/일반 Unix 우선 확인 파일
- `/etc/passwd`
- `/etc/shadow`

### 플랫폼별 추가 확인 파일
- `/etc/security/passwd`
  - AIX 계열 비밀번호 저장 위치
- `/tcb/files/auth`
  - HP-UX Trusted Mode 관련 경로
