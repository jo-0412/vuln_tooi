# U-06 Restrict su Command Usage to Authorized Accounts

## 1. Overview

- **Item Code**: U-06
- **Item Name**: Restrict su Command Usage to Authorized Accounts
- **Severity**: High
- **Category**: UNIX > Account Management

This item checks whether the use of the `su` command is restricted to a specific authorized group.

---

## 2. Why This Item Matters

The `su` command allows a logged-in user to switch to another user account.
If the target account is `root`, the user may obtain administrative privileges.

If all ordinary users are allowed to execute `su`, the following risks may arise:

- uncontrolled account switching
- unauthorized access to another user's environment
- increased attempts to obtain root privileges
- difficulty in tracking account misuse

The purpose of this item is to **restrict the use of the `su` command to authorized accounts only**.

---

## 3. Purpose of the Check

To restrict the use of the `su` command to a specific group
and prevent uncontrolled account switching and unauthorized root privilege escalation by ordinary users.

---

## 4. Files to Check

- `/etc/group`
- `/etc/pam.d/su`
- `/usr/bin/su`

---

## 5. Commands to Check

```bash
cat /etc/group | grep wheel
grep -v '^#' /etc/pam.d/su | grep pam_wheel
ls -l /usr/bin/su
