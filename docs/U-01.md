# U-01 root 계정 원격 접속 제한

## 1. 개요

- **항목 코드**: U-01
- **항목명**: root 계정 원격 접속 제한
- **중요도**: 상
- **분류**: UNIX > 계정 관리

이 항목은 시스템 정책에 **root 계정의 원격 터미널 직접 접속 차단 설정이 적용되어 있는지** 점검한다.

---

## 2. 왜 이 항목이 필요한가

root 계정은 시스템의 모든 기능을 설정·변경할 수 있는 최고 권한 계정이다.  
따라서 root 계정으로 외부에서 직접 원격 접속이 가능하면, 다음과 같은 위험이 커진다.

- 비밀번호 추측 공격
- 무차별 대입 공격(Brute Force)
- 사전 대입 공격(Dictionary Attack)
- 최고 권한 탈취 후 시스템 장악
- 감사 추적 및 사고 대응 난이도 증가

즉, 이 항목의 목적은 **외부 비인가자의 root 계정 직접 접근 시도를 원천 차단**하는 것이다.

---

## 3. Debian 계열에서 주로 확인할 파일

### 핵심 파일
- `/etc/ssh/sshd_config`
  - SSH 원격 접속 시 root 직접 로그인 허용 여부 확인
  - 주요 설정 키: `PermitRootLogin`

### Telnet 계열 확인 시 보조 파일
- `/etc/pam.d/login`
  - `pam_securetty` 적용 여부 확인
- `/etc/securetty`
  - `pts/*` 허용 항목 존재 여부 확인

### 참고
- Ubuntu 20.04 이상 및 일부 최신 배포판에서는 `/etc/securetty` 파일이 없을 수 있다.
- 최신 Debian/Ubuntu 환경에서는 보통 Telnet 서비스가 기본 비활성화 상태다.
- 따라서 **Debian 계열 1차 구현은 SSH 중심으로 판정**하고,
  Telnet 계열은 서비스 사용 시 추가 점검하는 구조가 적절하다.

---

## 4. 점검 목적

외부 비인가자가 root 계정으로 직접 원격 접속하여 시스템을 장악하는 것을 차단한다.

---

## 5. 점검 기준

### 양호
- 원격 터미널 서비스를 사용하지 않는 경우
- 원격 터미널 서비스를 사용하더라도 root 직접 접속을 차단한 경우

### 취약
- 원격 터미널 서비스를 사용하면서 root 직접 접속을 허용한 경우

---

## 6. Debian 계열 기준의 실제 판정 포인트

### 6-1. SSH 사용 시
`/etc/ssh/sshd_config` 파일에서 다음 값을 확인한다.

- `PermitRootLogin no`  
  → 양호

- `PermitRootLogin yes`  
  → 취약

- `PermitRootLogin` 값이 없거나 해석이 애매한 경우  
  → 자동 판정 보류 또는 수동 확인 필요

### 6-2. Telnet 계열 사용 시
다음 두 조건을 모두 만족해야 한다.

1. `/etc/pam.d/login` 에 `pam_securetty` 적용
2. `/etc/securetty` 에 `pts/*` 항목이 없어야 함

다음 중 하나라도 해당하면 취약으로 본다.

- `pam_securetty` 미적용
- `/etc/securetty` 에 `pts/0`, `pts/1` 등 원격 터미널 허용 항목 존재

---

## 7. 수집해야 하는 증적

도구는 최소한 아래 증적을 남겨야 한다.

### SSH 관련
- `/etc/ssh/sshd_config` 내 `PermitRootLogin` 설정값
- 설정 라인 원문
- SSH 서비스 활성 여부

### Telnet 관련
- `/etc/pam.d/login` 내 `pam_securetty` 적용 여부
- `/etc/securetty` 내 `pts/*` 존재 여부
- Telnet 또는 inetd/xinetd 계열 서비스 사용 여부

---

## 8. 사용자에게 보여줄 문제 설명

현재 서버는 root 계정의 원격 직접 로그인을 허용하고 있어,  
비밀번호 추측 또는 무차별 대입 공격 성공 시 즉시 최고권한 탈취로 이어질 수 있다.

---

## 9. 조치 방법

### SSH 사용 시
`/etc/ssh/sshd_config` 파일에서 다음과 같이 설정한다.

```conf
PermitRootLogin no
