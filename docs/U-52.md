# U-52 Disable Telnet Service

## 1. Overview

- **Item Code**: U-52
- **Item Name**: Disable Telnet Service
- **Severity**: High
- **Category**: UNIX > Service Management

This item checks whether the Telnet service is disabled.

---

## 2. Purpose

The purpose of this check is to prevent account credentials and sensitive information
from being exposed through plaintext Telnet communication.

Telnet does not encrypt usernames, passwords, or session data.
If Telnet is used on a network, credentials can be captured through packet sniffing.

---

## 3. Files to Check

- `/etc/inetd.conf`
- `/etc/xinetd.d/telnet`
- systemd Telnet service or socket units

---

## 4. Commands to Check

```bash
grep telnet /etc/inetd.conf
grep -i disable /etc/xinetd.d/telnet 2>/dev/null
systemctl list-units --type=service | grep telnet
systemctl list-units --type=socket | grep telnet
