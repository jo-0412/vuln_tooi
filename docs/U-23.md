# U-23 SUID, SGID, Sticky bit 설정 파일 점검

## 1. 개요

- **항목 코드**: U-23
- **항목명**: SUID, SGID, Sticky bit 설정 파일 점검
- **중요도**: 상
- **분류**: UNIX > 파일 및 디렉터리 관리

이 항목은 전체 파일시스템 내 특수권한(SUID, SGID, Sticky bit) 설정 항목을 식별하고,
불필요한 권한이 부여된 파일 또는 디렉터리가 존재하는지 점검하는 항목이다.

---

## 2. 왜 이 항목이 필요한가

SUID 또는 SGID가 설정된 실행 파일은 일반 사용자가 실행하더라도
파일 소유자 또는 그룹 권한으로 동작할 수 있다.

이 경우 다음과 같은 위험이 발생할 수 있다.

- 권한 상승(Privilege Escalation)
- 취약한 실행 파일 악용
- root 권한 우회 획득
- 그룹 권한 오남용
- 악성코드 또는 우회 스크립트 실행 통로 제공

Sticky bit는 주로 공유 디렉터리에서 삭제 제한 용도로 사용되지만,
비표준 위치에 존재하면 운영 목적을 별도로 검토해야 한다.

이 항목의 목적은 **불필요한 특수권한 파일을 제거하고 최소 권한 원칙을 적용하는 것**이다.

---

## 3. 점검 목적

불필요한 특수권한 파일을 제거해 권한 상승을 방지하기 위함

---

## 4. 확인 대상

고정 설정 파일이 아니라 **전체 파일시스템 내 특수권한 파일 및 디렉터리**를 점검한다.

대표적으로 확인하는 항목은 다음과 같다.

- SUID 설정 파일
- SGID 설정 파일
- Sticky bit 설정 디렉터리

---

find / -user root -type f \( -perm -04000 -o -perm -02000 \) -xdev -exec ls -al {} \;
