# U-02 비밀번호 관리정책 설정

## 1. 개요

- **항목 코드**: U-02
- **항목명**: 비밀번호 관리정책 설정
- **중요도**: 상
- **분류**: UNIX > 계정 관리

이 항목은 시스템에 비밀번호 복잡성, 변경 주기, 재사용 금지 정책이 적절히 설정되어 있는지 점검한다.

---

## 2. 왜 이 항목이 필요한가

비밀번호 관련 정책이 없거나 약하면 다음과 같은 문제가 발생할 수 있다.

- 짧고 단순한 비밀번호 사용 가능
- 영문/숫자/특수문자 조합이 없는 약한 비밀번호 허용
- 동일하거나 유사한 비밀번호 반복 사용 가능
- 무차별 대입 공격, 사전 대입 공격에 취약
- 계정 탈취 시도 성공 가능성 증가

이 항목의 목적은 **비밀번호 복잡성과 주기적 변경을 강제하여 계정 보안 수준을 높이는 것**이다.

---

## 3. Debian 계열에서 주로 확인할 파일

### 핵심 파일
- `/etc/security/pwquality.conf`
  - 비밀번호 복잡성 정책 확인
  - 최소 길이, 숫자/대문자/소문자/특수문자 요구 여부 확인

- `/etc/pam.d/common-password`
  - PAM을 통해 비밀번호 정책이 실제로 적용되는지 확인
  - `pam_pwquality.so`, `pam_pwhistory.so`, `pam_unix.so` 적용 여부 및 순서 확인

- `/etc/login.defs`
  - 비밀번호 최소/최대 사용기간 확인

### 보조 파일
- `/etc/security/pwhistory.conf`
  - 최근 비밀번호 재사용 금지 횟수 확인
- `/etc/pam.d/system-auth`
  - Debian이 아닌 계열과의 호환용 참고 경로
- `/etc/default/passwd`
- `/etc/default/security`

---

## 4. 점검 목적

비밀번호 복잡도와 변경 주기를 강제하여 계정 보안 수준을 높인다.

---

## 5. 점검 기준

### 양호
- 비밀번호 관리정책이 설정된 경우

### 취약
- 비밀번호 관리정책이 설정되지 않은 경우

---

## 6. Debian 계열 기준의 실제 판정 포인트

Debian 계열에서는 보통 아래 세 영역을 함께 본다.

### 6-1. 복잡성 정책
`/etc/security/pwquality.conf` 파일에서 다음 항목을 확인한다.

- `minlen = 8`
- `dcredit = -1`
- `ucredit = -1`
- `lcredit = -1`
- `ocredit = -1`
- `enforce_for_root`

#### 의미
- `minlen = 8`  
  최소 비밀번호 길이 8자 이상

- `dcredit = -1`  
  숫자 최소 1자 이상 요구

- `ucredit = -1`  
  대문자 최소 1자 이상 요구

- `lcredit = -1`  
  소문자 최소 1자 이상 요구

- `ocredit = -1`  
  특수문자 최소 1자 이상 요구

- `enforce_for_root`  
  root 계정에도 동일한 비밀번호 정책 적용

### 6-2. 재사용 금지 정책
최근 비밀번호 기억 횟수는 보통 다음 중 하나에서 확인한다.

- `/etc/security/pwhistory.conf`
- `/etc/pam.d/common-password`

권고 기준:
- `remember = 4` 이상

즉 최근 4회 이상 사용한 비밀번호를 다시 사용할 수 없도록 설정되어야 한다.

### 6-3. 변경 주기 정책
`/etc/login.defs`에서 다음 값을 확인한다.

- `PASS_MIN_DAYS 1`
- `PASS_MAX_DAYS 90`

#### 의미
- `PASS_MIN_DAYS 1`  
  비밀번호 변경 후 최소 1일은 사용해야 다시 변경 가능

- `PASS_MAX_DAYS 90`  
  비밀번호는 최대 90일까지 유효

### 6-4. PAM 적용 여부 및 순서
`/etc/pam.d/common-password`에서 다음 모듈을 확인한다.

- `pam_pwquality.so`
- `pam_pwhistory.so`
- `pam_unix.so`

중요한 점:
- `pam_pwquality.so`
- `pam_pwhistory.so`

이 두 모듈은 **`pam_unix.so`보다 위에 위치해야 정책이 적용된다.**

또한 가이드 기준으로는  
`/etc/security/pwquality.conf` 또는 `/etc/pam.d/common-password`(또는 `system-auth`) 중 어느 하나라도 비밀번호 관리 정책이 설정되어 있으면 양호로 볼 수 있다. 다만 도구 구현에서는 더 보수적으로 **복잡성 + 주기 + 재사용 금지 + PAM 적용 상태**를 함께 보는 것이 좋다.

## 7. 수집해야 하는 증적

도구는 최소한 아래 정보를 증적으로 남겨야 한다.

### 복잡성 관련
- `minlen`
- `dcredit`
- `ucredit`
- `lcredit`
- `ocredit`
- `enforce_for_root`

### 재사용 금지 관련
- `remember` 값

### 변경 주기 관련
- `PASS_MIN_DAYS`
- `PASS_MAX_DAYS`

### PAM 적용 관련
- `pam_pwquality.so` 존재 여부
- `pam_pwhistory.so` 존재 여부
- `pam_unix.so` 존재 여부
- 각 모듈의 실제 위치 순서
- 관련 설정 라인 원문

## 8. 사용자에게 보여줄 문제 설명

비밀번호 정책이 약하거나 없어서 짧고 단순한 비밀번호 사용이 가능하며,  
동일 또는 유사 비밀번호 재사용으로 계정 탈취 위험이 높다.
